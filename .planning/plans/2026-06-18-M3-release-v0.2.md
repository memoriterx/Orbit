# M3 — Release v0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump orbit-base from 0.1.0 to 0.2.0 across every distribution manifest, author a Keep-a-Changelog `CHANGELOG.md` covering OMC-1~4, and create a local `v0.2.0` git tag — stopping before any irreversible external publish.

**Architecture:** This is a release-engineering task, not a feature task. The "surface" is four JSON manifests that each carry an independent `version` string; they must move together or the marketplace will report a version mismatch. A new `CHANGELOG.md` becomes the human-facing record. A git tag marks the commit. Push / GitHub release / marketplace publish are split into a separate, explicitly-gated final section because they are irreversible (OMC-4 trigger T1).

**Tech Stack:** JSON manifests, Markdown (Keep a Changelog 1.1.0 format), git, `python3 -m json.tool` (JSON validity, matches the SubagentStop gate), `grep` (version-consistency assertions), `jq` (optional cross-check).

## Global Constraints

- Release scope is **`plugins/orbit-base/` + distribution manifests only.** The dev-team `.claude/` tree is NOT a release artifact and MUST NOT be version-bumped.
- Domain purity: `grep -rE 'oremi|Oremi' plugins/orbit-base/` MUST return 0 hits (SubagentStop gate blocks otherwise).
- Commit prefixes: `chore:` for the version bump, `docs:` for the CHANGELOG. (OMC convention.)
- NO Co-Authored-By line in any commit.
- Target version is exactly `0.2.0`; git tag is exactly `v0.2.0` (semver minor bump — new backward-compatible features OMC-1~4; no breaking change ⇒ not 1.0.0; more than a patch ⇒ not 0.1.1).
- The four manifest `version` fields and the tag must all read the same version. This equality is the release's primary success criterion.
- External publish (push / GitHub release / marketplace) is OUT of the default scope. It runs only on explicit, separate user approval (Task 5).
